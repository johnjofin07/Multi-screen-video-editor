# screen reference

Use this file when you need extra app-specific context while generating UI.

## editor screen priorities
1. Preview first
2. Playback controls second
3. Layout choice third
4. Simple timeline fourth
5. Clip detail actions in a bottom sheet or compact section
6. Export always easy to find

## timeline reference
The timeline should feel approachable:
- one horizontal strip is preferred in V1
- show clip blocks with rounded rectangles
- offsets should be readable without exposing too much editor jargon
- use small labels like "Start", "Offset", "Trim"
- do not create a dense professional NLE timeline

## layout selector reference
Use compact visual chips or cards for:
- side by side
- top and bottom
- pip
- grid

The selected option should be obvious, but avoid colorful states.

## picker entry reference
This screen should feel light and easy:
- short copy
- one strong CTA
- minimal supporting text
- remind the user they can choose 2 to 4 videos

## export flow reference
The export flow should feel calm and predictable:
- preset selection should be simple
- file naming should be obvious
- progress should reduce anxiety
- success should lead to open/share quickly
